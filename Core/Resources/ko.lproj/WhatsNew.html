Growl 2.1의 새로운 점

주요 변화

  • 알림 동작에 스크립트 실행 기능 추가
  • 규칙 기반으로 알림을 제어할 수 있는 기능이 추가되었습니다. 다음 링크 http://growl.info/documentation/applescript-rules를 방분하시면 자세한 정보를 확인하실 수 있습니다.
  • 100개가 넘은 많은 알림을 동시에 받을 때도 더 이상 멈추지 않습니다.
  • Webkit 알림 모양도 원하는 모양의 닫기 버튼을 만들 수 있습니다.

일반 변경 사항

  • 응용 프로그램이 알림의 우선순위를 결정해야 될 때를 정확하게 했습니다.
  • AppleScript 이미지 크기가 128x128에서 1024x1024로 커졌습니다.
  • 통신을 통한 알림 전달 기능과 OS X의 알림 센터 기능을 같이 사용할 때, 양쪽 컴퓨터에서 모두 알림 내용을 볼 수 있습니다.
  • 알림을 눌렀을 때의 동작이 설정되어있을떄 도 Prowl을 통한 알림 전달이 올바르게 작동되게 하였습니다.

Automator

  • Automator 동작에 대한 내용이 번역되었습니다.
  • GrowlAction 작업 흐름이 Automator의 설명 부분에 나옵니다.

Growl 2 변경 사항

일반

  • 10.8에서 OS X 알림 센터를 지원합니다.
  • Prowl 과 Boxcar 서비스를 알림 동작으로 추가하였습니다.
  • 모아 보기에서 자리 비움 감지 기능이 향상되었습니다.
  • Growl 이 샌드박스 처리되었습니다.

환경 설정

  • Growl에서 사용하는 켜기/끄기 스위치가 더욱 이해하기 편해졌습니다.

알림 모양

  • 말하기 기능에 대한 상세한 조절이 가능해졌습니다.
  • 알림 모양과 알림 동작 기능을 구분하였습니다.
  • Music Video 에서 글자 글자 정렬을 왼쪽 정렬과 오른쪽 정렬을 할 수 있습니다.
  • Music Video 에서 여러 줄의 알림이 나타날 때 높이 문제를 수정하였습니다.
  • Nano 알림은 왼쪽, 오른쪽, 가운데의 여러곳에서 나타날 수 있습니다.
  • SoundAction 알림을 추가하였습니다.
  • 알림 모양을 사용하지 않을 수도 있게 하였습니다.

버그 수정

  • CPU 점유율이 높게 나타나던 문제를 수정하였습니다.
  • 알림이 여러 줄일 때 충돌이 나던 문제를 수정하였습니다.
  • 단축키를 해제할 수 있게 되었습니다.

Growl 1.4 변경 사항

일반

  • 응용 프로그램 탭이 새롭게 디자인되었습니다! 새롭게 작성된 중요한 작업이며, 사용하기 더욱 쉬워졌습니다.
  • 알림 모아 보기 기능이 개별 탭으로 옮겨졌습니다.
  • 통신에 관한 기능이 개선되었습니다.

환경 설정 패널 재정비

  • 켬/끔 스위치에 새로운 슬라이더 스위치
  • 위치 지정시 키보드에 대한 지원 추가
  • 일반적인 탭 재구성
  • 통신 탭에 구독 기능 추가
  • 알림 기록 탭에서 개별 응용 프로그램과 알림 설정으로 연결
  • 모아 보기 탭에서 모아 보기 창 관련 설정
  • 새로운 알림 모양 추가시 Growl이 재시작 하지 않아도 되도록 수정

통신

  • GNTP에 URL 필드를 추가하여 '알림 콜백 타겟' 기능을 지원합니다. 알림을 클릭하면 OSX가 URL을 처리하게됩니다. (예를 들어 http:// 주소의 경우 기본 웹브라우저가 처리합니다.)
  • 다른 컴퓨터에서 알림을 받을 수 있는 구독 기능이 지원됩니다.
  • 네트워크 인증 기능이 보안되었습니다. 비밀번호 항목이 빈 칸인 경우 비밀번호 헤더가 필요 없도록 하였습니다.
  • 주소 데이터 해석에 관계된 네트워크 충돌을 수정하였습니다.
  • 외부에서 들어오는 알림을 받지 않는 경우는 로컬 소켓에만 바인딩 하도록 했습니다.
  • 주소 데이터 캐싱과 캐쉬를 비우기 위한 네트워크 변화 감시

Applescript

  • GNTP의 '알림 콜백 타켓'과 같은 URL 클릭 피드백 지원

버그 수정

  • 웹킷 추가 기능 생성에 관계된 충돌을 수정하였습니다.
  • 알림 전달에 관계된 충돌을 수정하였습니다.
  • 말하기 알림에서 이전 메시지의 종료를 기다리는 동안 CPU를 잡아먹는 문제를 수정하였습니다.
  • 웹킷 모양 스타일에서 CFErrorGetDomain과 관계된 충돌을 수정하였습니다.
  • 재설치시 추가 기능 목록이 중복으로 나타나는 문제를 수정하였습니다.
  • '알림 콜백 타겟' 기능이 메소드 헤더가 없더라도 고려되고 사용됩니다.
  • 손상된 알림 기록 데이터베이스에 관한 문제를 처리하였습니다.
  • 알림 기록 목록에서 하나의 항목을 삭제할 경우 정렬과 관계된 문제를 수정하였습니다.
